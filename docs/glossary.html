An attribute is a value that describes items. In tabular data, for example, columns are synonmous with attributes.

An attribute that describes whether or not two items are the same, at least with respect to the relevant characteristic. It usually does not make sense to perform math operations on categorical such as adding, and the values do not have a natural ordering.

Individual nodes or edges are of the same class when they have the same real-world meaning and/or the same set of attributes. For example, in a movie network, "People" and "Movies"could be different node classes. Edges can also have separate classes, such as "Cast" versus "Crew" edges.

When clusters exist in data, items can be grouped based on their attributes.

Color images typically have multiple components per pixel, such as red, green, blue and/or alpha.

A column represents an attribute in a tabular dataset.

In a network, a cycle exists if there is a route along edges from any node back to itself. Hierarchy datasets do not contain cycles.

One or more special characters in textual data that divide the data in a way that a computer can recognize the divisions. For example, commas in a comma-separated list act as a delimiter.

One of the possible directions in which spatial data can be positioned. Dimensions can correspond to real spatial directions, time, or imaginary or abstract directions.

Edge direction implies the possibility of movement or an ordered relationship between the nodes on either end. For example, edges between parent nodes and child nodes in a hierarchy could be considered to have direction.

A set of rules that describe the order in which characters and words should appear. A formal language can be simple as a genetic sequence of four characters (e.g. ACATTAG), or as complex as computer code.

A segment of temporal data, or spatial data with a single dimension, that has a beginning point and an ending point.

A set of points and/or shapes with one dimensions, suspended in a space with two dimensions.
